Bagaimana cara pakai API ini?

Alur gampang: (1) siapkan header standar; (2) kalau perlu token, login dulu; (3) panggil endpoint sesuai method; (4) cek respons & tangani error.
Header Standar
  • Accept: application/json
  • Content-Type: application/json (untuk request ber-body)
  • Authorization: Bearer <TOKEN> (kalau endpoint butuh token)
Checklist Cepat
  • Butuh token? Panggil POST /api/login ‚Üí simpan token.
  • Gunakan nilai category yang konsisten (makanan / minuman).
  • Cek status code untuk memastikan hasil.

üîê Login POST

Tujuan: Autentikasi user. Jika sukses, gunakan token-nya pada request selanjutnya.
Endpoint
POST https://krusit-production.up.railway.app/api/login
Body (JSON)
{
  "email": "EMAIL_KAMU",
  "password": "PASSWORD_KAMU"
}
Contoh cURL
curl -X POST "https://krusit-production.up.railway.app/api/login"   -H "Accept: application/json"   -H "Content-Type: application/json"   -d '{"email":"EMAIL_KAMU","password":"PASSWORD_KAMU"}'
Contoh fetch()
fetch("https://krusit-production.up.railway.app/api/login", {
  method: "POST",
  headers: {
    "Accept": "application/json",
    "Content-Type": "application/json"
  },
  body: JSON.stringify({ email: "EMAIL_KAMU", password: "PASSWORD_KAMU" })
}).then(r => r.json()).then(console.log)

Status & Penanganan Error

  • 200 OK / 201 Created ‚Äî Berhasil.
  • 400 / 422 ‚Äî Validasi gagal. Cek field & tipe data.
  • 401 / 403 ‚Äî Token salah/kadaluarsa atau tidak berizin.
  • 404 ‚Äî Endpoint/ID tidak ditemukan, cek path {id}.
  • 500 ‚Äî Error server. Cek log server & body request.
